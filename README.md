# 网页活动 AI 工作流

这个仓库用于沉淀网页活动产品经理在网页活动 / H5 活动从 0-1 落地过程中的 AI 工作流，包括提示词、输出模板、Codex Skill 和检查清单。

它不是泛泛介绍 AI 工具的资料库，而是面向产品经理日常工作的使用仓库。适用场景包括：需求沟通、会议记录整理、需求澄清、同步文档检查、原型与页面逻辑生成、埋点设计、提测前自查、LQA 多语言检查、压测信息收集、域名与对外发布检查、活动复盘。

## 推荐使用方式

1. 日常处理活动需求时，优先从 `提示词_prompts` 中选择对应环节的提示词，复制到 ChatGPT 或其他 AI 工具中使用。
2. 如果要生成最终可交付内容，结合 `输出模板_templates` 统一文档结构，减少返工和遗漏。
3. 遇到高频、稳定、规则明确的任务时，沉淀到 `技能_skills`，用于后续 Codex Skill 化。
4. 长期业务背景、网页活动 SOP 摘要、个人工作方式和常见约束，统一放到 `业务背景_context`，作为后续提示词和 Skill 的上下文来源。

## 各目录用途

`提示词_prompts`：存放日常可复制使用的提示词，覆盖会议整理、需求澄清、同步文档检查、页面逻辑、埋点、提测、LQA、压测、发布和复盘等环节。

`技能_skills`：沉淀高频、稳定、规则明确的 Codex Skill，例如埋点设计、同步文档检查、上线风险检查。Skill 应比普通提示词更强调输入要求、输出格式、禁止事项和检查规则。

`输出模板_templates`：统一最终产物格式，便于复制到同步文档、PRD、群消息、复盘文档或检查清单中。

`业务背景_context`：存放长期业务背景、网页活动 SOP 摘要、个人工作背景和常见协作信息。这里的信息用于辅助 AI 理解场景，但不应被擅自扩写成新的业务规则。

## 使用原则

- 信息不足时，明确标记“待补充”。
- 检查类输出要区分“已确认 / 待确认 / 风险项 / 建议动作”。
- 所有内容尽量写成产品文档风格：清晰、可执行、少废话。
- 不要把人的流程要求误写成 AI 能自动执行的动作。
- 不要擅自新增没有事实依据的业务规则。
