# 上线风险检查清单

## 已确认

- [ ] inner 回归已完成。
- [ ] 阻塞问题已关闭或有明确处理方案。
- [ ] 域名、path、地区域名已确认。
- [ ] 宣发链接已确认。
- [ ] 对外时间已确认。
- [ ] 运维已预约。
- [ ] 相关人力已预留。
- [ ] 看板安排已确认。
- [ ] 流量观察安排已确认。
- [ ] 下线时间和下线方式已确认。

## 待确认

- [ ] 对外时间是否早于宣发。
- [ ] 是否存在链接提前外泄风险。
- [ ] 是否需要地区域名。
- [ ] 是否需要额外运维值守。
- [ ] 是否需要发布后第二天搭建看板。

## 风险项

- [ ] inner 回归未完成。
- [ ] 域名、path 或宣发链接未确认。
- [ ] 对外时间早于宣发时间。
- [ ] 运维未预约。
- [ ] 上线观察人力不足。
- [ ] 看板未安排。
- [ ] 下线信息未确认。

## 建议动作

- [ ] 补齐所有待确认项后再同步最终对外信息。
- [ ] 将域名、path、宣发链接统一给需求方确认。
- [ ] 提前预约运维并确认观察窗口。
- [ ] 明确上线后流量观察负责人。
- [ ] 确认活动结束后的下线动作和责任人。
