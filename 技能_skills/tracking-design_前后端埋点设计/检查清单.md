# 前后端埋点设计检查清单

## 0. 后端需求类型确认

- [ ] 用户是否已明确说明本次是后端配置需求还是后端定制开发需求。
- [ ] 如果用户未说明后端类型，是否先向用户确认，而不是自行判断。
- [ ] 如果是后端配置需求，是否没有生成后端埋点。
- [ ] 如果是后端定制开发需求，是否生成了必要的后端埋点。
- [ ] 是否避免把后台配置奖励、概率、库存、任务、活动时间等内容误写成后端埋点。
- [ ] 前端埋点是否仍然完整生成。

## 1. 基础信息确认

- [ ] 已提供活动名称或活动 ID。
- [ ] 已提供页面列表。
- [ ] 已提供用户主流程。
- [ ] 已提供页面模块和核心交互。
- [ ] 已提供看板关注指标。
- [ ] 已提供历史命名规则或确认无历史规则。

## 2. 前端埋点完整性

- [ ] 是否覆盖页面访问。
- [ ] 是否覆盖页面模块曝光。
- [ ] 是否覆盖按钮点击。
- [ ] 是否覆盖弹窗展示。
- [ ] 是否覆盖 tab 切换。
- [ ] 是否覆盖分享点击。
- [ ] 是否覆盖规则入口点击。
- [ ] 是否覆盖奖励说明入口点击。
- [ ] 是否覆盖任务入口点击。
- [ ] 是否覆盖抽奖入口点击。
- [ ] 是否覆盖兑换入口点击。
- [ ] 是否覆盖商城商品点击。
- [ ] 是否覆盖主要 CTA 点击。

## 3. 后端埋点必要性

- [ ] 后端埋点是否只在后端定制开发需求下生成。
- [ ] 后端埋点是否只覆盖定制开发相关业务结果。
- [ ] 是否没有为后台配置奖励、概率、库存、任务、活动时间、开关、礼包、道具、活动规则生成后端埋点。
- [ ] 是否没有因为活动涉及奖励、抽奖、兑换、任务就自动生成后端埋点。

## 4. 后端埋点范围

- [ ] 是否覆盖新增接口的关键业务结果。
- [ ] 是否覆盖定制任务完成结果。
- [ ] 是否覆盖定制奖励发放结果。
- [ ] 是否覆盖定制抽奖结果。
- [ ] 是否覆盖定制兑换结果。
- [ ] 是否覆盖定制礼包发放结果。
- [ ] 是否覆盖定制支付 / 下单 / 购买结果。
- [ ] 是否覆盖定制邀请关系确认结果。
- [ ] 是否覆盖定制资格校验结果。
- [ ] 不适用项是否未强行生成。

## 5. 命名与参数

- [ ] 事件名称是否沿用历史格式。
- [ ] 页面名称是否和产品文档一致。
- [ ] 上报结构是否合并成一行。
- [ ] 上报结构是否满足看板分析。
- [ ] 是否避免输出玩家 ID、订单 ID、设备 ID 等过细参数。
- [ ] 是否避免输出失败原因枚举等偏后端实现的内容。

## 6. 风险项

- [ ] 用户未确认后端类型，但输出了后端埋点。
- [ ] 后端配置需求中出现后端埋点。
- [ ] 后台配置项被误写成后端定制开发结果埋点。
- [ ] 事件过多，包含无意义埋点。
- [ ] 固定上报结构被拆成多行重复写。
- [ ] 未按历史命名规则输出。

## 7. 建议动作

- [ ] 后端类型未确认时，先向用户提问确认。
- [ ] 删除无法支撑看板分析的埋点。
- [ ] 删除后端配置需求下的后端埋点。
- [ ] 合并固定上报结构。
- [ ] 将缺失命名规则标记为“待补充”。
- [ ] 和数据/看板使用方确认核心指标口径。
