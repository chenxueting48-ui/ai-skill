# 同步文档补全检查清单

## 输出结构

- [ ] 是否按 sheet 分组输出。
- [ ] 是否覆盖活动基础信息、投放链接、原型&设计稿、页面多语言、活动任务、抽奖概率、用户中奖信息收集、埋点、后端告警、宣发计划、验收反馈、需求变更、每周进度同步。
- [ ] 是否区分可填写 / 待确认 / 不适用。
- [ ] 是否生成整体缺失信息。
- [ ] 是否生成风险项。
- [ ] 是否生成需要同步给需求方 / 研发 / 测试 / 运维的信息。
- [ ] 是否生成可直接发群里的同步话术。

## 关键字段

- [ ] 是否没有脑补奖励、概率、库存、道具 ID、奖励号。
- [ ] 是否明确时间和时区。
- [ ] 时间是否明确到时分秒和时区；缺失时是否标记待确认。
- [ ] 是否提示 LQA 周期风险。
- [ ] 是否提示页面截图、字符限制、时间格式和时区展示风险。
- [ ] 是否提示压测所需 PV/UV、峰值、QPS。
- [ ] 是否没有脑补 QPS。
- [ ] 是否提示域名和 path 需要确认。
- [ ] 是否没有破坏投放链接公式。

## 埋点处理

- [ ] 是否正确处理前后端埋点。
- [ ] 前端埋点是否仍然提示需要生成。
- [ ] 是否确认后端需求类型。
- [ ] 如果后端类型未确认，是否没有生成后端埋点。
- [ ] 如果是后端配置需求，是否没有生成后端埋点。
- [ ] 如果是后端定制开发需求，是否只生成定制开发相关后端埋点。

## 禁止事项

- [ ] 是否没有输出不对应同步文档字段的散文式 PRD。
- [ ] 是否没有直接生成没有依据的上线结论。
- [ ] 是否没有把后台配置需求误写成后端定制开发需求。
- [ ] 是否没有把 LQA 返回内容或最终翻译写成已确认。
- [ ] 是否没有把验收反馈处理结论写成已完成。
