# 同步文档补全 Skill

## name

sync-doc-fill

## description

当用户需要根据活动需求、会议记录、群聊结论、原型说明、页面文案、奖励规则、任务规则、埋点需求或宣发信息，生成可填入活动同步文档的内容时使用。

## when_to_use

在以下场景使用：

- 用户需要创建新的活动同步文档；
- 用户需要根据需求信息补全同步文档；
- 用户需要把散乱信息整理到固定 sheet 中；
- 用户需要把会议结论转成同步文档字段；
- 用户需要检查哪些字段可以填、哪些字段必须待确认。

## required_inputs

尽量提供：

- 活动名称；
- 游戏简称；
- 活动目标；
- 活动时间；
- 时区；
- 活动地址；
- 页面语言；
- 分享渠道；
- 页面结构；
- 页面文案；
- 任务规则；
- 奖励规则；
- 概率；
- 库存；
- 道具 ID；
- 奖励号；
- 宣发计划；
- 埋点规则；
- 是否存在后端定制开发；
- 是否涉及 LQA / 压测 / 域名 / inner 回归 / 看板。

## output_format

固定按以下结构输出：

```text
# 同步文档补全结果

## 一、整体缺失信息

列出会影响同步文档补全的关键信息。

## 二、按 sheet 补全内容

### 1. 活动基础信息

| 字段 | 建议填写内容 | 状态 | 备注 |
| -- | ------ | -- | -- |

状态只能使用：

* 可填写
* 待确认
* 不适用

每个 sheet 都按同样结构输出。

## 三、风险项

按高 / 中 / 低风险输出。

## 四、需要同步给需求方 / 研发 / 测试 / 运维的信息

按对象分组输出。

## 五、可直接发群里的同步话术

生成一段简短、清晰、可直接复制到群里的话术。
```

## rules

必须遵守：

- 同步文档是最终信息沉淀载体；
- 输出必须按同步文档 sheet 分组；
- 可以根据原始材料生成草稿，但不能脑补关键业务字段；
- 活动时间必须明确到时分秒和时区，缺失则标记待确认；
- 奖励、概率、库存、道具 ID、奖励号不能脑补；
- QPS 不能脑补，只能在 PV/UV、峰值集中时间足够时计算或提示缺失；
- 投放链接不主动改公式，只提示需要填写基础链接和渠道；
- 页面多语言可以生成文案草稿，但 LQA 返回内容和最终翻译不能脑补；
- 埋点内容需要遵守前后端埋点 Skill 的规则；
- 后端埋点只有用户明确说明“后端定制开发需求”时才生成；
- 如果只是后端配置需求，不生成后端埋点；
- 如果用户没有说明后端类型，必须先向用户确认。

## forbidden_outputs

禁止：

- 脑补奖励概率；
- 脑补库存；
- 脑补道具 ID；
- 脑补奖励号；
- 脑补活动时间和时区；
- 脑补 QPS；
- 直接生成没有依据的上线结论；
- 把后台配置需求误写成后端定制开发需求；
- 在未确认后端类型时生成后端埋点；
- 输出不对应同步文档字段的散文式 PRD。

## examples

### 示例输入

活动为新用户任务抽奖 H5，页面包含首页、任务模块、抽奖弹窗。已有中文文案和任务方向；奖励概率、库存、活动时间、时区、宣发量级、域名 path、后端类型均未确认。

### 示例输出

```text
# 同步文档补全结果

## 一、整体缺失信息
- 活动时间和时区待确认。
- 奖励概率、库存、道具 ID、奖励号待确认。
- 宣发 PV / UV、峰值集中时间待确认。
- 域名和 path 待确认。
- 后端需求类型待确认。

## 二、按 sheet 补全内容

### 1. 活动基础信息
| 字段 | 建议填写内容 | 状态 | 备注 |
| -- | ------ | -- | -- |
| 活动目标 | 新用户任务抽奖 | 可填写 | 基于输入材料整理 |
| 活动时间 | 待确认 | 待确认 | 需明确到时分秒和时区 |

### 6. 抽奖概率
| 字段 | 建议填写内容 | 状态 | 备注 |
| -- | ------ | -- | -- |
| 中奖概率 | 待确认 | 待确认 | 不可脑补 |
| 库存 | 待确认 | 待确认 | 不可脑补 |
```
