# 同步文档检查清单

## 1. sheet 覆盖

- [ ] 是否检查活动基础信息。
- [ ] 是否检查投放链接。
- [ ] 是否检查原型&设计稿。
- [ ] 是否检查页面多语言。
- [ ] 是否检查活动任务。
- [ ] 是否检查抽奖概率。
- [ ] 是否检查用户中奖信息收集（选填）。
- [ ] 是否检查埋点（前端+后端）。
- [ ] 是否检查后端告警。
- [ ] 是否检查宣发计划。
- [ ] 是否检查验收反馈收集表。
- [ ] 是否检查需求变更申请表。
- [ ] 是否检查每周进度和问题同步模板（待确认）。

## 2. 输出结构

- [ ] 是否按 sheet 分组输出。
- [ ] 是否区分已确认 / 待确认 / 风险项 / 建议动作。
- [ ] 是否有整体结论。
- [ ] 是否有风险项汇总。
- [ ] 是否有建议动作汇总。
- [ ] 是否有可直接发群里的同步话术。

## 3. 关键字段检查

- [ ] 活动时间是否明确到时分秒和时区。
- [ ] 活动地址、域名、path 是否确认。
- [ ] 投放链接基础链接、渠道、参数是否确认。
- [ ] 是否没有破坏投放链接公式。
- [ ] 原型和设计稿链接是否确认。
- [ ] 页面文案、页面截图、字符限制是否确认。
- [ ] LQA 翻译和测试周期是否确认。
- [ ] 任务名称、完成条件、周期、次数、奖励是否确认。
- [ ] 奖励、概率、库存、道具 ID、奖励号是否确认。
- [ ] 用户中奖信息收集是否明确是否适用。
- [ ] 后端告警指标、阈值、负责人是否确认。
- [ ] 宣发渠道、宣发时间、PV/UV、峰值集中时间是否确认。
- [ ] 验收反馈是否有处理状态和负责人。
- [ ] 需求变更是否有影响范围和确认方。

## 4. 压测与上线风险

- [ ] 是否提示压测所需 PV/UV、峰值、QPS。
- [ ] 是否没有脑补 QPS。
- [ ] 是否提示域名和 path 需要确认。
- [ ] 是否提示对外时间和宣发时间关系。
- [ ] 是否提示 inner 回归、看板和上线观察信息缺失。

## 5. 埋点检查

- [ ] 是否正确处理前后端埋点。
- [ ] 前端埋点是否仍然完整检查。
- [ ] 是否检查后端需求类型。
- [ ] 如果后端类型未确认，是否没有判断需要后端埋点。
- [ ] 如果是后端配置需求，是否没有生成或要求后端埋点。
- [ ] 如果是后端定制开发需求，是否只检查定制开发相关后端埋点。

## 6. 禁止事项

- [ ] 是否没有脑补奖励、概率、库存、道具 ID、奖励号。
- [ ] 是否没有脑补活动时间和时区。
- [ ] 是否没有脑补 QPS。
- [ ] 是否没有替需求方、研发、测试、运维确认结论。
- [ ] 是否没有直接生成“可以上线”的结论。
- [ ] 是否没有输出不对应同步文档字段的散文式检查结果。
