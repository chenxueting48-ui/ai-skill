# 原型与页面逻辑生成

## 使用场景

根据活动玩法和需求信息，生成页面列表、用户主流程、页面模块、按钮/弹窗/状态逻辑、异常态、空态、未登录态、活动未开始/已结束态、后台配置依赖和需求侧确认项。

## 输入信息

- 活动目标
- 核心玩法
- 用户路径
- 页面或模块草稿
- 奖励、任务、时间、登录态等已知规则
- 后台配置能力或限制

## 输出要求

- 生成页面结构和主流程。
- 梳理每个页面的模块、按钮、弹窗和状态。
- 标记后台配置依赖和待确认项。
- 不把缺失信息写成确定逻辑。

## 输出格式

```text
# 原型与页面逻辑

## 页面列表
## 用户主流程
## 页面模块说明
## 按钮/弹窗/状态逻辑
## 异常态/空态
## 未登录态
## 活动未开始/已结束态
## 后台配置依赖
## 需求侧确认项
```

## 检查规则

- 是否覆盖用户进入、参与、完成、失败或异常路径。
- 是否明确登录态和活动时间状态。
- 是否列出后台配置依赖。
- 是否把缺失规则标记为“待补充”。

## 禁止事项

- 不要擅自新增活动玩法。
- 不要替设计输出视觉方案。
- 不要把后台能力未确认的逻辑写成可实现。
- 不要输出和输入无关的页面。

## 示例输入

用户登录后完成每日任务可获得抽奖机会，活动有首页、任务列表和中奖弹窗。任务规则待补充。

## 示例输出

```text
## 页面列表
- 首页
- 任务列表
- 中奖弹窗

## 用户主流程
进入活动页 -> 登录 -> 查看任务 -> 完成任务 -> 获得抽奖机会 -> 抽奖 -> 查看中奖结果。

## 需求侧确认项
- 每日任务类型和完成口径待补充。
- 抽奖机会是否可累积待补充。
```
