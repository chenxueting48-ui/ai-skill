# 同步文档补全

## 使用场景

当用户提供活动需求、会议记录、群聊结论、原型说明、页面文案、奖励规则、任务规则、埋点需求、宣发信息等材料时，用于根据活动同步文档模板输出可复制到对应 sheet 的内容。

## 输入信息

- 活动需求或背景说明
- 会议记录、群聊结论或私聊结论
- 原型说明和页面结构
- 页面文案和多语言范围
- 奖励规则、任务规则、概率、库存
- 道具 ID、奖励号
- 埋点需求和看板关注指标
- 宣发计划、PV / UV、峰值集中时间
- 活动时间、时区、活动地址、域名和 path
- 是否涉及 LQA / 压测 / inner 回归 / 看板
- 是否存在后端定制开发

## 输出要求

- 按同步文档 sheet 分组输出。
- 每个 sheet 输出“可填写内容 / 待确认内容 / 风险提醒”。
- 不要生成原文没有依据的奖励、概率、库存、道具 ID、奖励号、时间、时区、QPS。
- 对缺失信息明确标记“待确认”。
- 所有输出偏产品文档风格，清晰、可复制、少废话。

## 检查规则

- 是否按 sheet 分组输出。
- 是否区分“可填写内容 / 待确认内容 / 风险提醒”。
- 是否没有脑补奖励、概率、库存、道具 ID、奖励号。
- 是否没有脑补活动时间、时区、QPS。
- 是否提示 LQA、压测、域名、path、inner 回归、看板相关缺失信息。
- 是否按前后端埋点规则处理后端类型。
- 是否没有破坏投放链接公式。

## 禁止事项

- 不要脑补奖励概率、库存、道具 ID、奖励号。
- 不要脑补活动时间和时区。
- 不要脑补 QPS。
- 不要直接生成没有依据的上线结论。
- 不要把后台配置需求误写成后端定制开发需求。
- 不要在未确认后端类型时生成后端埋点。
- 不要输出不对应同步文档字段的散文式 PRD。

## 示例输入

活动为新用户任务抽奖 H5，页面包含首页、任务模块、抽奖弹窗。需求方提供了中文文案和任务方向，但奖励概率、库存、活动时间、时区、宣发量级、域名 path、后端类型均未确认。

## 示例输出

```text

- 活动时间和时区待确认。
- 奖励概率、库存、道具 ID、奖励号待确认。
- 宣发 PV / UV、峰值集中时间待确认。
- 域名和 path 待确认。
- 后端需求类型待确认。

## 二、按 sheet 补全内容

### 1. 活动基础信息
#### 可填写内容
- 活动类型：新用户任务抽奖 H5。
- 页面结构：首页、任务模块、抽奖弹窗。

#### 待确认内容
- 活动上线时间、结束时间、时区。
- 活动地址、域名、path。

#### 风险提醒
- 时间和时区缺失会影响配置、测试、LQA 和宣发。
```
