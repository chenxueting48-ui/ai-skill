# 前后端埋点设计生成

## 使用场景

根据网页活动 / H5 活动的页面、用户路径、核心交互、看板关注指标和后端需求类型，生成前端埋点和必要的后端埋点设计。

适用于：

- 活动页面和核心交互已经基本明确；
- 需要生成可复制到同步文档“埋点（前端+后端）”sheet 的内容；
- 需要判断本次是否需要后端埋点；
- 需要检查埋点是否存在无意义事件、过细参数或后端类型误判。

## 输入信息

- 活动名称或活动 ID
- 页面列表
- 用户主路径
- 页面模块和核心交互
- 看板关注指标
- 已有埋点命名规则
- 历史埋点格式
- 后端需求类型：后端配置需求 / 后端定制开发需求 / 未说明
- 后端定制开发范围：仅在后端定制开发需求时提供

## 输出要求

- 前端埋点无论如何都需要生成。
- 后端埋点只有用户明确说明“后端定制开发需求”时才生成。
- 如果用户明确说明是“后端配置需求”，则不生成后端埋点。
- 如果用户未说明后端类型，必须先向用户确认，不要自行判断。
- 不要因为活动涉及奖励、任务、抽奖、兑换，就自动生成后端埋点。
- 不要把后台配置项误写成后端定制开发项。
- 不要输出玩家 ID、订单 ID、设备 ID、失败原因枚举、后端错误码细分等看板不用的细参数。
- 固定上报结构合并成一行。
- 备注简短，只写必要说明。

## 检查规则

- 是否已确认后端需求类型。
- 如果未确认后端类型，是否先提问而不是生成后端埋点。
- 如果是后端配置需求，是否只生成前端埋点。
- 如果是后端定制开发需求，是否只生成定制开发相关后端业务结果埋点。
- 前端埋点是否覆盖页面访问、模块曝光、按钮点击、弹窗展示、tab 切换、分享点击、规则入口、奖励说明入口、任务入口、抽奖入口、兑换入口、商城商品点击、主要 CTA 点击。
- 是否避免输出无意义埋点。
- 是否避免玩家 ID、订单 ID、设备 ID、失败原因枚举、后端错误码细分等过细参数。
- 是否沿用用户提供的命名规则和历史格式。

## 禁止事项

- 不要在用户未明确后端类型时，自行判断后端类型。
- 不要在用户未明确后端类型时，直接生成后端埋点。
- 不要在只有后端配置需求时生成后端埋点。
- 不要把后台配置项误写成后端埋点。
- 不要因为活动涉及奖励、任务、抽奖、兑换，就默认生成后端埋点。
- 不要把后端配置后台已有能力写成后端定制开发结果埋点。
- 不要跳过后端类型确认，直接输出完整埋点表。
- 不要输出玩家 ID、订单 ID、设备 ID、失败原因枚举、后端错误码细分等看板不用的细参数。
- 不要把固定上报结构拆成多行重复写。
- 不要擅自改变已有命名规则。

## 示例输入

活动包含首页、任务模块、抽奖模块和奖励弹窗。看板关注首页访问、任务入口点击、抽奖入口点击、奖励弹窗展示。用户说明本次奖励、概率、库存、任务均由已有活动后台配置完成，没有后端定制开发。
